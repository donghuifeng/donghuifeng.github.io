浏览器内核

全世界主流的浏览器一共有五个，分别是IE、Firefox、Chrome、Safari、Opera，划分的基础在于他们拥有独立的内核，其内核分别为trident、gecko、webkit/blink、webkit、presto（新版本的Opera也用webkit）。至于国内的百度、360、猎豹、QQ等其他浏览器，都是没有独立内核的。

HTML负责实现网页结构，CSS负责实现网页样式，JavaScript负责实现网页行为.

四张图片并列问题

凡是带有inline的元素，都有文字属性；文字属性的意思是会根据空格进行分割，这样在分行编写的img代码就会出现图片之间隔开的现象。

绝对定位

层模型

absolute绝对定位的元素会脱离原来位置进行定位，上升到上一层，本身的位置会被其他元素占据；每一个absolute元素都是一个新的层；

relative相对定位

元素是保留原来位置进行定位，上升到上一层，但是本身的位置也不让其他元素占据；

一般都是relative用来做基准，absolute进行定位操作

最近的有定位的父级元素进行定位，如果没有，那么相对于文档进行定位

相对文档定位，就是相对网页左上角定位

margin塌陷

子元素margin-top相对于父元素不起作用，只有当子元素的值超过父元素的margin-top值的时候，子元素带着父元素一起动了；解决方式一是给父元素加一个border-top，不用了

解决方式二让父级变成bfc ，英文全称是block format context；每一个盒子里面都有一套相同的语法规则 可以通过bfc让某个盒子的渲染规则发生变化。如何触发一个盒子的bfc?常见的有四种方式，分别是position:absolute; display:inline-block; float:left/right;overflow:hidden;用哪个bfc方式解决比较好，要根据实际需要选择。

margin合并问题

指的是出现区域不能共用的情况；第一种解决办法是通过bfc的方式，但是破坏了html的结构，不推荐采用；第二种方式就是不解决，我们通过数学的办法来解决；

float浮动模型

浮动元素产生了浮动流

所有产生了浮动流的元素，块级元素看不到他们。产生了bfc的元素和文本类属性的元素以及文本都能看到浮动元素，其中文本类元素指的是带有inline属性的元素。

父级元素包不住子元素，因为父级元素是块级元素，子元素是浮动元素，所以看不到子元素；解决方法一人为设置高度，失败；解决方法二，是清除浮动流，clear有三个属性值，分别是left, right和both。

清除浮动流的方法一，但是结构的问题，不能通过修改结构来解决，所以第一种方案不好

伪元素

存在于任意一个元素里面；任意一个元素里面有两个最特殊的伪元素，一个叫before，一个叫after；伪元素和正常元素一样可以正常来使用，但是没有正常元素结构，可以被css选中和操作。

伪元素天生是行级元素。加入content伪元素才能生效；

清除浮动流方法二，用伪元素来处理，其中clear需要在块级元素里面才起作用，所以还需要把伪元素变成块级元素；

父级元素也变成bfc，就可以看到浮动流，达到包含的目的
